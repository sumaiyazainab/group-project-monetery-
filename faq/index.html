    Frequently Asked Questions

    Staging and Validation

    What UniProt accession ID format should I use?

    Use the standard UniProt accession ID format (e.g. P05067, Q9Y2T1), which can be found by searching for your protein on the UniProt website. Gene names, entry names, or isoform identifiers are not accepted.

    Do I need to select a start codon when starting a new experiment?

    Selecting the start codon that corresponds to your expression system may reduce computational runtime. ATG is the canonical start codon in most systems. GTG and TTG are alternative start codons found in prokaryotes such as E. coli. If unsure, leave all three unselected and the system will automatically consider all three during ORF identification.

    The experimental staging succeeded, but the alignment score is low. Is this concerning?

    A score above 0.85 is accepted as a valid match. Any score between 0.85 and 0.95 indicates minor sequence differences between the canonical UniProt reference protein and the protein encoded in the plasmid. This might be expected if the plasmid encodes a mutated or engineered variant of the wild-type protein.

    My alignment score is very low and experimental staging failed. What does this mean?

    A score below 0.85 is not accepted as a valid match. This may be due to an incorrect UniProt accession ID, an incorrect plasmid FASTA, a mismatched start codon selection, or a highly divergent sequence. Please verify all of these before retrying.

    Data Upload and Quality Control

    Could the system support other file formats?

    No — the portal currently only accepts FASTA files for plasmid upload, and TSV or JSON files for experimental data. However, since the parsing module uses Pandas, extending support to other formats such as CSV or Excel could be added in future versions if required.

    Can I upload data from multiple generations in one file?

    Yes — the portal expects a single TSV or JSON file containing all generations. The system automatically groups variants by generation using the Directed_Evolution_Generation column.

    Can I upload experimental data with additional metadata columns?

    The parser is designed with a flexible schema, so additional metadata columns do not cause errors or affect analysis. Only the seven essential fields are validated and used for analysis, while any extra columns are preserved in the database for potential future use.

    What happens to rows that fail quality control?

    Rows with missing values in essential fields are excluded from analysis and are not stored in the database. The remaining dataset proceeds to variant activity analysis normally.

    Activity Score and Results

    What does an Activity Score of 1.0 mean?

    A score of 1.0 means the variant performed equivalently to the wildtype control for that generation. Scores above 1.0 indicate improved catalytic activity, and scores below 1.0 indicate reduced activity relative to the wildtype control.

    Why would a variant with a high Activity Score have low protein abundance?

    The Activity Score is calculated as DNA yield divided by protein yield, normalised to the wildtype control mean. A variant with low protein expression but proportionally high DNA output will score highly, reflecting efficient catalytic activity per unit of protein produced rather than high expression.

    What does clustering in the 3D PCA landscape mean?

    Clustering of high-activity variants in the same region of the landscape suggests those variants share similar sequence features associated with improved performance. This can help identify which regions of sequence space are worth targeting in future mutagenesis rounds.
